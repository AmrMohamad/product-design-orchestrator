# Playbook — Feature from request to production

1. Inspect the requested journey and affected system.
2. Translate the request into user problem, outcome, scope, evidence, assumptions, and consequences.
3. Decide whether discovery/research is needed; do not run a study when existing evidence is sufficient.
4. Map the whole flow, including entry, prerequisites, data, permission, alternates, recovery, completion, and downstream service impact.
5. Write screen/component contracts and an approved visual/interaction direction.
6. Prototype only risky/uncertain behavior; validate with realistic tasks when consequence warrants.
7. Plan vertical slices and implement using existing tokens/primitives/architecture.
8. Complete loading, empty, partial, error, stale, offline, permission, destructive, interrupted, and success states.
9. Render and verify all relevant viewports, themes, locales, roles, content extremes, keyboard/touch, and performance.
10. Release with outcome/guardrail instrumentation, owner, and review trigger.
