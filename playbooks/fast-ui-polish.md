# Playbook — Fast UI polish without slop

Use only when product structure and behavior are already sound.

1. Render the real UI with representative content/states at target widths.
2. Write the one-sentence intended character and preserve list.
3. Audit hierarchy, typography, composition, spacing rhythm, density, color/material, imagery/icons, controls, and motion.
4. Identify generic defaults and the weakness they mask; do not swap clichés mechanically.
5. Choose one high-leverage signature move and keep the rest disciplined.
6. Correct structure and type before decorative detail; make optical adjustments in browser.
7. Check awkward widths, long/short content, dark/light, focus, errors, loading, empty, and reduced motion.
8. Compare before/after against the task and thesis; remove additions that do not earn their cost.
9. Run focused accessibility/performance/visual regression checks.
10. Record system-level fixes so quality does not remain a one-off patch.
