# Design Systems Skills

6 bundled skills.

## `accessibility-by-design`

Translate accessibility and inclusive user needs into flow, component, content, token, and acceptance requirements before implementation.

## `component-system`

Specify reusable components with purpose, anatomy, variants, states, accessibility, content, tokens, API boundaries, tests, and misuse guidance.

## `design-system-migration`

Plan and execute migration from fragmented or legacy UI to a coherent system without breaking product behavior or freezing delivery.

## `design-tokens`

Define platform-agnostic primitive, semantic, and component design tokens with themes, aliases, descriptions, governance, and code translation.

## `documentation-governance`

Document, govern, evolve, and measure adoption of a design system through clear ownership, contribution, review, versioning, migration, and deprecation.

## `theming`

Design semantic theme architecture for light/dark, high contrast, brand, density, and platform modes with predictable inheritance and asset behavior.
