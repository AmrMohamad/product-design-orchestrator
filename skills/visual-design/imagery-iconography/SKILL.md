# Imagery Iconography

## Mission

Use visual assets to explain, orient, and create character rather than fill empty rectangles.

## Use this skill when

- images
- icons
- illustration system
- art direction
- visual assets

## Inputs

- brand/aesthetic thesis
- content/jobs
- asset inventory/rights
- platform/performance/accessibility constraints

## Protocol

1. Define the job of each asset category: evidence, explanation, identity, navigation, emotion, or decoration.
2. Choose art direction: subject, framing, lighting, color, crop, background, texture, and treatment consistency.
3. Inventory existing assets and usage rights; avoid fabricated endorsements or misleading imagery.
4. Choose one icon family and define size, stroke/fill, optical box, labels, state, and custom-icon criteria.
5. Create responsive crop/aspect/focal rules and low-bandwidth/failed-media behavior.
6. Write meaningful alt text for informative images and empty alt for decoration; provide long/structural alternatives for complex diagrams.
7. Optimize formats, dimensions, loading, and layout reservation.
8. Test dark theme, high contrast, localization, and real content pairings.

## Required outputs

- asset/art-direction system
- icon rules
- responsive/media behavior
- accessibility text strategy
- performance/rights notes

## Quality bar

- Assets have a named purpose.
- One coherent visual family is used.
- Rights/provenance are clear.
- Media failure does not break the task.

## Failure modes to avoid

- Stock-photo filler.
- Mixed icon families.
- Icons without labels when meaning is ambiguous.
- AI imagery presented as documentary truth.
- Unreserved image space.

## Handoff

Record consequential decisions and pass only decision-relevant context to the next phase.
