# Jobs To Be Done

## Mission

Explain why someone changes behavior and what successful progress means in context.

## Use this skill when

- JTBD
- jobs to be done
- user motivation
- outcome statement

## Inputs

- interviews/context evidence
- current journey
- alternatives/workarounds
- decision scope

## Protocol

1. Identify a concrete switching or struggle episode rather than abstract preference.
2. Map situation/trigger, desired progress, functional/social/emotional dimensions, and constraints.
3. Identify push of current situation, pull of new approach, anxiety, and habit/inertia.
4. Document alternatives people hire today and the criteria used to judge them.
5. Write job stories grounded in evidence: When…, I want to…, so I can….
6. Translate to desired outcomes and opportunity areas without prescribing a feature.
7. Check segment/context differences and contradictory jobs.
8. Use the model to prioritize and test concepts, not as decorative workshop language.

## Required outputs

- job model
- forces/alternatives
- job stories
- desired outcomes
- design implications

## Quality bar

- Jobs come from real episodes.
- Social/emotional dimensions are not invented.
- Alternatives include non-products.
- The model changes prioritization or concept criteria.

## Failure modes to avoid

- Writing a task as a job.
- Generic universal jobs.
- Using JTBD instead of usability evidence.
- Invented forces.

## Handoff

Record consequential decisions and pass only decision-relevant context to the next phase.
