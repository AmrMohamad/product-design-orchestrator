# Changelog

## 0.4.1 — 2026-06-25 (beta)

- Added a local-only, cross-platform, receipt-driven self-installer for Codex and Claude Code with project/user scope, dry-run, verification, idempotent updates, conflict protection, and reversible uninstall.
- Added a generated flat plugin bundle with valid `.codex-plugin/plugin.json` and `.claude-plugin/plugin.json` manifests, all 69 existing skills, a `pdo-start` gateway, and self-contained governance/resources.
- Added official local marketplace catalogs for both platforms and documented native marketplace installation alternatives.
- Added a Claude Code `CLAUDE.md` interoperability bridge while retaining `AGENTS.md` as the shared operating contract.
- Added `pdo-*` Codex skill names to avoid replacing unrelated installed skills, plus a context-light orchestrated profile.
- Added Bash, PowerShell, and CMD wrappers, installer smoke tests, platform research notes, and expanded package validation.
- Preserved the v4.0 design methods, skill content, playbooks, registries, templates, and authority model unchanged.

## 0.4.0 — 2026-06-25 (beta)

- Rebuilt the former 5,000-line monolithic guide as a 955-line controlling orchestrator plus modular expertise.
- Added a complete, proportional product/service lifecycle from orientation and discovery through live learning and retirement.
- Added evidence labels, confidence handling, anti-fabrication research rules, inclusion/safety, and robust desk/field research protocols.
- Added contextual human-made taste: Design Read, six design dials, aesthetic thesis, creative tension, signature move, reference decomposition, and anti-default discipline.
- Added same-agent design-to-code responsibility, stack inspection, vertical slices, complete states, and deterministic pixel-accuracy verification.
- Added 67 original specialist skills, 2 preserved MIT-licensed upstream references, and a searchable registry covering all 110 researched UI Skills entries plus the Taste Skill family.
- Added safe Markdown-only skill acquisition, project initialization, package validation, nine playbooks, sixteen Markdown templates, token example, sources, research synthesis, and prior-skill crosswalk.
- Added `AGENTS.md` and `SOUL.md` with an explicit authority model: `DESIGN.md` controls behavior and evidence; `SOUL.md` controls professional presence.
