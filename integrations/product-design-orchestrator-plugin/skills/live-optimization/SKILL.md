# Live Optimization

## Mission

Continue product design after launch by diagnosing real behavior and testing the most valuable correction.

## Use this skill when

- improve live product
- post launch UX
- continuous optimization
- analytics design

## Inputs

- live metrics/telemetry
- support/search/error logs
- research/feedback
- release history
- product/system roadmap

## Protocol

1. Compare current outcome/guardrails with baseline and expected mechanism; segment by relevant context and access barriers.
2. Identify where the journey breaks through funnels, paths, errors, latency, support, qualitative evidence, and accessibility feedback.
3. Distinguish instrumentation defects, seasonality, cohort mix, product change, and genuine behavior.
4. Create problem hypotheses and seek why through targeted research before prescribing UI changes.
5. Prioritize by user consequence, strategic value, confidence, effort, and reversibility.
6. Choose direct improvement, experiment, research, or system fix; define success and guardrails.
7. Release incrementally, monitor adverse effects, and document learning including null results.
8. Update design system, content, support, and evidence artifacts so learning compounds.

## Required outputs

- live diagnosis
- prioritized hypotheses
- research/experiment/change plan
- result/learning record
- system/backlog updates

## Quality bar

- Optimization ties to user outcomes.
- Qualitative and quantitative evidence complement each other.
- Segments and excluded users are considered.
- Learning persists beyond the dashboard.

## Failure modes to avoid

- Chasing conversion alone.
- Reacting to one comment.
- UI tweaks without causal hypothesis.
- Never removing ineffective features.

## Handoff

Record consequential decisions and pass only decision-relevant context to the next phase.
