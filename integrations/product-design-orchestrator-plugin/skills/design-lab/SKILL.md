# Design Lab

## Mission

Turn subjective visual uncertainty into a structured experiment without generating a wall of arbitrary alternatives.

## Use this skill when

- explore variants
- design lab
- show directions
- iterate visual design

## Inputs

- design question
- approved brief/principles
- base content/flow
- rendering environment
- feedback criteria

## Protocol

1. Name the visual or interaction decision being explored and freeze unrelated variables.
2. Ask at most one focused question if two directions depend on missing intent; otherwise proceed with explicit assumptions.
3. Create 2–4 variants that differ on one meaningful dimension or coherent thesis.
4. Render variants with identical realistic content, viewport, and state.
5. Annotate each with intention, trade-off, system implications, and likely failure mode.
6. Gather feedback against predeclared criteria and ask what evidence drove the response.
7. Combine only compatible strengths; avoid averaging every direction into blandness.
8. Record selection and update tokens/components rather than leaving one-off styling.

## Required outputs

- experiment frame
- rendered variants
- trade-off annotations
- feedback synthesis
- selected system update

## Quality bar

- Variants answer a real question.
- Unrelated variables stay controlled.
- Evaluation uses criteria.
- The selected result is integrated systemically.

## Failure modes to avoid

- Twenty random variants.
- Color swaps called concepts.
- Popularity voting without rationale.
- Averaging away the thesis.

## Handoff

Record consequential decisions and pass only decision-relevant context to the next phase.
