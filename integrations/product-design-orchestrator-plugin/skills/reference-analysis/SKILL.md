# Reference Analysis

## Mission

Extract the hidden design decisions that make a reference work and decide which belong in the current product.

## Use this skill when

- analyze reference
- inspiration
- screenshot design analysis
- style decomposition

## Inputs

- reference images/URLs
- current product brief
- target platform
- brand/IP constraints

## Protocol

1. Identify the reference surface, audience, task, state, and capture conditions.
2. Trace reading order, focal hierarchy, grid, alignment, negative space, and density.
3. Measure or estimate type roles/metrics, color roles, material, borders/shadows, imagery/icon treatment, and motion grammar.
4. Separate system rules from one-off campaign art and content effects.
5. Explain what creates perceived quality: proportion, contrast, rhythm, reduction, specificity, or interaction—not brand prestige.
6. List transferable principles, context-dependent choices, and elements that would become imitation.
7. Compare multiple references to find a unique synthesis.
8. For pixel rebuilds, create a formal reference contract and measurements.

## Required outputs

- reference decomposition
- transfer/do-not-copy table
- measurement notes
- synthesized principles
- reference contract when needed

## Quality bar

- Analysis names mechanisms, not adjectives.
- Capture conditions/unknowns are documented.
- IP/brand boundaries are respected.
- Principles fit the target task.

## Failure modes to avoid

- "Apple-like" as a specification.
- Copying assets/content.
- Ignoring typography/content.
- Treating a static screenshot as complete behavior.

## Handoff

Record consequential decisions and pass only decision-relevant context to the next phase.
