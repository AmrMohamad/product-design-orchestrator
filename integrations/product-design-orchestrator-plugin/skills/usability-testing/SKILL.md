# Usability Testing

## Mission

Observe behavior against explicit task and risk criteria rather than ask whether participants like the design.

## Use this skill when

- usability test
- test prototype
- moderated test
- task testing

## Inputs

- research questions
- prototype/product and fidelity limits
- participant criteria
- task scenarios
- risk/accessibility needs

## Protocol

1. Define which decisions and failure modes the test can credibly answer at the current fidelity.
2. Write realistic goal-based scenarios that do not reveal interface labels or steps.
3. Prepare neutral moderation, consent, think-aloud guidance, recovery rules, and observer protocol.
4. Pilot tasks, data/state, timing, recording, accessibility, and technical setup.
5. Observe path, hesitation, misunderstanding, errors, recovery, confidence, and system feedback; distinguish moderator help.
6. Debrief after behavior, not before; ask comprehension and expectation questions.
7. Analyze severity by task impact, persistence, recovery, affected context, and frequency within sample.
8. Separate prototype limitations from product findings.
9. Decide fix, investigate, accept, or retest; verify critical corrections.

## Required outputs

- test plan
- scenarios/moderator guide
- observation log
- severity findings
- design decisions/retest plan

## Quality bar

- Tasks reflect real goals.
- Behavior outweighs preference.
- Fidelity limitations are explicit.
- Critical accessibility and recovery paths are included.

## Failure modes to avoid

- Leading participants to success.
- Counting completion after moderator rescue.
- Testing only the happy path.
- Calling a small qualitative sample statistically representative.

## Handoff

Record consequential decisions and pass only decision-relevant context to the next phase.
