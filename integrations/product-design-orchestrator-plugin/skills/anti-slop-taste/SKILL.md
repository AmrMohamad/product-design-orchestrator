# Anti Slop Taste

## Mission

Replace model defaults with a coherent authored thesis, real content, disciplined composition, and selective detail.

## Use this skill when

- anti slop
- make UI feel human
- generic AI UI
- polish visual design

## Inputs

- design read and dials
- rendered UI
- brand/system context
- user/task/risk

## Protocol

1. State what the product should feel like and why before changing style.
2. Audit for generic defaults: gradient/glow, centered hero, equal cards, bento without hierarchy, over-rounding, pill overload, icon soup, fake data, and motion everywhere.
3. Identify the underlying weakness each cliché is masking: weak content, no hierarchy, no concept, no imagery, poor spacing, or absent state design.
4. Choose one signature move and simplify surrounding surfaces.
5. Strengthen typography, composition, density, content specificity, imagery/material, and interaction feedback in that order.
6. Introduce asymmetry or novelty only where it improves hierarchy/character and remains responsive.
7. Perform optical and content-extreme corrections in the browser.
8. Compare before/after against the aesthetic thesis and task—not trendiness.

## Required outputs

- anti-default audit
- visual thesis/signature move
- prioritized corrections
- rendered before/after review

## Quality bar

- The result belongs to the product.
- Familiar patterns remain when useful.
- Distinctiveness does not harm access or speed.
- Real states/content feel authored.

## Failure modes to avoid

- Replacing one cliché with another.
- Banning common patterns categorically.
- Arbitrary chaos.
- Decorating before fixing hierarchy.
- Using expensive motion as taste.

## Handoff

Record consequential decisions and pass only decision-relevant context to the next phase.
