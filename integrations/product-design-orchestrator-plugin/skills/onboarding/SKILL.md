# Onboarding

## Mission

Help a new or returning user reach meaningful value with the least forced ceremony and enough context to succeed.

## Use this skill when

- onboarding
- first run
- activation
- setup flow

## Inputs

- activation outcome
- user segments/readiness
- required setup/data/permissions
- support and lifecycle

## Protocol

1. Define the earliest meaningful success and what must truly precede it.
2. Separate mandatory setup, optional personalization, education, permissions, migration, and team/invite work.
3. Use progressive disclosure and contextual teaching; avoid explaining features before need arises.
4. Allow skip, defer, back, resume, reset, and support where safe.
5. Design zero-data/empty states that lead to the next valuable action.
6. Explain permission/data requests at the moment of value and allow denial/recovery.
7. Support different entry points, imported/existing data, invited users, returning users, and role differences.
8. Measure task/value attainment and later success, not checklist completion alone.
9. Research confusion, trust, abandonment, and support burden after launch.

## Required outputs

- activation model
- onboarding flow/states
- education/content plan
- permission/recovery behavior
- metrics/research plan

## Quality bar

- Users can reach value without a generic tour.
- Required steps are truly required.
- Progress persists.
- Activation metrics connect to sustained value.

## Failure modes to avoid

- Carousel tours.
- Forced profile completion without value.
- Celebrating setup instead of outcome.
- Permission prompts without context.

## Handoff

Record consequential decisions and pass only decision-relevant context to the next phase.
